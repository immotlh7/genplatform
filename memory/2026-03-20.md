## Self-Development System (Critical Feature)

Med requested the most critical page in GenPlatform - a self-development system for autonomous platform development.

### Architecture: Two-Agent System
1. **Orchestrator Agent** (Claude Sonnet in Next.js):
   - Reads uploaded task files (e.g., PRIORITY-1.md with 100+ tasks)
   - Decomposes into ultra-precise micro-tasks
   - Manages 200K context by sending only focused tasks
   - Monitors execution progress
   - Never executes code directly

2. **Developer Agent** (Claude Opus on OpenClaw/Telegram):
   - Receives one micro-task at a time (200 tokens vs 4000)
   - Executes specific file changes
   - Reports completion back
   - Builds and tests every 5 tasks
   - Commits after each message completion

### Implementation:
1. Created WORK-RULES.md with protected files list
2. **Upload API** - `/api/self-dev/upload/route.ts`:
   - Accepts .md file uploads
   - Saves to /root/genplatform/data/task-files/
   
3. **Analyze API** - `/api/self-dev/analyze/route.ts`:
   - Calls Claude Sonnet to decompose tasks
   - Returns JSON with micro-tasks grouped in batches
   - Each task has: filePath, description, specificChanges
   
4. **Execute API** - `/api/self-dev/execute/route.ts`:
   - Sends individual tasks to Telegram bot
   - Formats as "MICRO-TASK: {id}" with clear instructions
   
5. **UI Components**:
   - FileUploader with drag-and-drop
   - ExecutionMonitor showing progress, batches, current tasks
   - Two-agent architecture explanation cards
   
6. **Self-Dev Page** - `/dashboard/self-dev/page.tsx`:
   - Complete workflow from upload → analyze → execute
   - Real-time progress tracking
   - Stats display (messages, tasks, batches)

### Key Innovation:
This solves the context window problem - instead of sending entire task files to the Developer agent (using 50%+ of context), the Orchestrator digests everything and sends only precise micro-instructions. This keeps the Developer's context clean for actual coding.

### Security:
- Added protected files to WORK-RULES.md
- Self-dev system files are NEVER modifiable by future tasks
- Core layout files (sidebar, navbar, globals.css) protected

✅ Build successful, deployed, committed as "Self-Development System: Two-agent architecture for autonomous platform development"
